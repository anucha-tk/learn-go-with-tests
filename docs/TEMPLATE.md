# ชื่อเรื่อง

> Hook: ประโยคเปิดเรื่อง ทำให้คนอยากอ่านต่อ (1-2 บรรทัด)

## เรื่องของเรา

เล่า context เหมือนนั่งคุยกัน ใช้สรรพนาม "เรา/คุณ" แทน "the reader". ตั้งคำถามที่อยากตอบในบทนี้.

## ปัญหาแรกที่เจอ

เล่า pain point ก่อนจะรู้จัก concept. ทำให้คนเห็นว่า "เออ มันเจอจริง".

## ก่อนแก้ — แบบ naive

```go
// โค้ดแบบแรกที่เขียนตามสัญชาตญาณ
// มักจะยัง compile ไม่ผ่าน หรือทำงานได้แต่แย่
```

## หลังแก้ — แบบ idiomatic

```go
// โค้ดที่ "ถูกต้อง" ในแบบ Go
// เปรียบเทียบ diff ให้เห็นชัดว่าเปลี่ยนตรงไหน ทำไม
```

## ทำไมต้องรู้เรื่องนี้

- เหตุผลทางเทคนิค 2-3 ข้อ
- เชื่อมกับเรื่องอื่นใน Go ecosystem
- ผลกระทบถ้าไม่รู้ (เช่น bug, performance, readability)

## หลุมที่เจอบ่อย
